# Монетизация — «Универсальная мастерская файлов»

## 1. Модель монетизации

### Принцип: «Полезно бесплатно, удобнее за деньги»

```
Бесплатно:                          Платно:
─────────────────                   ─────────────────
Все базовые инструменты             Большие файлы
Ограниченный размер файла           Пакетная обработка
Ограниченное число операций/день    Больше операций
Нет истории                         История + повтор
Нет пакетной обработки              Приоритетная очередь
Реклама отсутствует                 Расширенные инструменты
                                    Длительное хранение
```

Ключевое: бесплатный пользователь ВСЕГДА получает результат. Мы не блокируем скачивание, не показываем результат за paywall. Ограничения — по объёму и удобству, не по доступу к базовым функциям.

### Почему это работает:

1. Пользователь приходит с конкретной задачей → решает её бесплатно → доверяет сервису
2. Возвращается повторно → упирается в лимит (5 операций/день) → регистрируется (10 операций)
3. Регулярно использует → упирается в размер файла или пакетную обработку → покупает Plus
4. Бизнес-пользователь → нужна команда, API, SLA → покупает Business

## 3. Что оставить бесплатно (навсегда)

Эти функции — точка входа. Они должны работать без трения:

- Конвертация изображений (JPG↔PNG↔WEBP, HEIC→JPG)
- Сжатие изображений
- Resize изображений
- Сжатие PDF
- Merge PDF (до 3 файлов)
- Split PDF
- DOCX → PDF
- Markdown → HTML
- Удаление EXIF
- Скачивание результата (без задержки, без рекламы)

Почему: это самые частые запросы. Пользователь приходит из поиска, решает задачу, запоминает сервис. Конверсия в регистрацию и оплату происходит через повторные визиты.

## 4. Что переводить в платное

### Plus (мотивация: «мне нужно больше»)

- Файлы > 25 МБ
- Более 15 операций в день
- Пакетная обработка (несколько файлов за раз)
- Расширенные PDF-инструменты (extract pages, reorder, rotate выбранных страниц)
- Расширенные image-инструменты (crop, batch resize)
- Пресеты (сохранённые настройки операций)
- История > 7 дней

### Pro (мотивация: «это мой рабочий инструмент»)

- Файлы > 100 МБ
- Массовая обработка (50 файлов)
- Все инструменты без ограничений
- Размеры для маркетплейсов (Wildberries, Ozon, Amazon)
- Водяной знак
- Favicon-пакет
- Длительное хранение (30 дней)

### Business (мотивация: «нужно для команды»)

- Несколько пользователей
- API доступ
- Рабочие пространства
- Файлы до 2 ГБ
- Безлимитные операции
- Приоритетная поддержка

## 6. Стимулирование регистрации

### Что получает пользователь при регистрации (бесплатно):

```
Без регистрации:          С регистрацией:
5 операций/день     →     15 операций/день
Нет истории         →     История 7 дней
Файлы 1 час        →     Файлы 24 часа
Нет повтора         →     Повтор операции
```

### Как показывать:

```
После 3-й операции (не раньше!):
┌─────────────────────────────────────────────┐
│ 💡 Зарегистрируйтесь бесплатно и получите: │
│    • 15 операций в день (сейчас 5)          │
│    • Историю операций                        │
│    • Повтор прошлых действий                 │
│                                              │
│    [Регистрация за 30 секунд]               │
└─────────────────────────────────────────────┘
```

Правила:
- Не показывать при первом визите
- Не показывать popup — только inline-блок под результатом
- Не блокировать скачивание
- Показывать после 3-й операции или при попытке использовать функцию, требующую регистрации
- Максимум 1 раз за сессию

## 7. Стимулирование оплаты без раздражения

### Принцип: показывать ценность, а не ограничение

```
ПЛОХО:  "Вы исчерпали лимит. Купите подписку."
ХОРОШО: "Файл 35 МБ. На тарифе Plus можно обрабатывать файлы до 100 МБ."

ПЛОХО:  "Эта функция доступна только в Pro."
ХОРОШО: "Пакетная обработка: загрузите до 10 файлов за раз. Доступно в Plus."

ПЛОХО:  Popup при каждом действии.
ХОРОШО: Тихая метка "Plus" рядом с функцией + страница /pricing.
```

### Точки конверсии (где показывать апгрейд):

| Момент | Что показать |
|--------|-------------|
| Файл > 25 МБ | "Файл X МБ. Plus: до 100 МБ" |
| 15-я операция за день | "Лимит на сегодня. Plus: 100 операций/день" |
| Попытка batch upload | "Пакетная обработка — в Plus" |
| В истории (7 дней) | "История хранится 7 дней. Plus: 90 дней" |
| На странице профиля | Виджет "Ваш тариф: Free" с кнопкой "Улучшить" |
| После 10-й операции за всё время | Inline-блок с преимуществами Plus |

### Чего НЕ делать:

- Popup при каждом визите
- Countdown timer ("осталось 2 операции!")
- Fake scarcity ("скидка только сегодня!")
- Скрытие кнопки скачивания
- Задержка скачивания ("подождите 30 секунд или купите")
- Реклама сторонних продуктов

## 8. Оплата: РФ и западный рынок

### Для РФ:
- ЮKassa (бывшая Яндекс.Касса) — карты, SBP, ЮMoney
- CloudPayments — карты, SBP
- Тинькофф Оплата
- Подписка через рекуррентные платежи

### Для западного рынка:
- Stripe — карты, Apple Pay, Google Pay
- Paddle — как merchant of record (упрощает налоги)
- LemonSqueezy — альтернатива Paddle

### Стратегия:
1. MVP: без оплаты. Все функции бесплатно с лимитами. Валидация спроса.
2. После 1000 регистраций: подключить оплату для РФ (ЮKassa)
3. После валидации конверсии: подключить Stripe для западного рынка
4. Цены адаптировать по регионам (PPP — purchasing power parity)

### Годовая подписка:
- Скидка ~17% от месячной (2 месяца бесплатно)
- Показывать как "экономия X ₽/год"
- По умолчанию показывать годовую цену (выше LTV)

## 9. Метрики для роста выручки

### Воронка:

```
Визит → Операция → Регистрация → Повторный визит → Оплата → Retention
```

### Ключевые метрики:

| Метрика | Формула | Цель |
|---------|---------|------|
| Activation Rate | Пользователи с ≥1 операцией / Все визиты | > 40% |
| Registration Rate | Регистрации / Пользователи с ≥3 операциями | > 20% |
| D7 Retention | Вернулись через 7 дней / Зарегистрированные | > 15% |
| D30 Retention | Вернулись через 30 дней / Зарегистрированные | > 8% |
| Free → Paid Conversion | Платные / Зарегистрированные | > 3% |
| Trial → Paid (если будет trial) | Оплатили после trial / Начали trial | > 25% |
| MRR | Сумма месячных подписок | Растёт |
| ARPU | MRR / Платные пользователи | > $5 |
| Churn Rate | Отменили подписку / Всего платных (в месяц) | < 5% |
| LTV | ARPU / Churn Rate | > $100 |
| CAC | Расходы на привлечение / Новые платные | < LTV/3 |

### Операционные метрики:

| Метрика | Зачем |
|---------|-------|
| Самые популярные инструменты | Куда вкладывать ресурсы |
| Среднее время обработки | Качество сервиса |
| % ошибок задач | Надёжность |
| Средний размер файла | Планирование инфраструктуры |
| Операций на пользователя в день | Engagement |
| Точки отказа (где уходят) | UX-проблемы |
| Конверсия по точкам апгрейда | Какие ограничения работают |

### Что отслеживать в первую очередь (MVP):

1. Количество операций в день (общее)
2. Топ-5 инструментов
3. Регистрации в день
4. % ошибок задач
5. Средний размер файла

Для аналитики на MVP достаточно: audit_logs + простой SQL-дашборд. Позже — Posthog или Plausible (privacy-friendly).

## 10. Roadmap монетизации

### Фаза 0: Валидация (0-3 месяца)
- Все функции бесплатно с лимитами
- Сбор метрик: какие инструменты используются, сколько операций
- Цель: 1000 регистраций, понять product-market fit

### Фаза 1: Первые деньги (3-6 месяцев)
- Подключить оплату (ЮKassa для РФ)
- Запустить Plus тариф
- Цель: 50 платных пользователей, валидация willingness to pay

### Фаза 2: Масштабирование (6-12 месяцев)
- Запустить Pro и Business тарифы
- Подключить Stripe для западного рынка
- Годовые подписки
- Цель: MRR $1000+

### Фаза 3: B2B (12+ месяцев)
- API доступ
- Рабочие пространства
- Размеры для маркетплейсов
- Цель: B2B-клиенты с ARPU > $50/мес
