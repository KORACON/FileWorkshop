# UI Improvement Audit — этап усиления интерфейса

## 1. Аудит проблем

### Палитра
- Текущая палитра (#F4F7FB / #2563EB) уже деловая, но ТЗ требует сдвиг к графитово-синему (#F3F6FA / #2E5B88). Нужно обновить primary и фоны.
- Поверхности карточек (#FFFFFF) слишком стерильные — нужен #FCFDFE.

### Header
- Нет профиль-dropdown. Сейчас: Link на /profile + кнопка "Выйти". Нужно: аватар-триггер → dropdown с аватаром/именем/почтой + пункты.
- Кнопка "Выйти" торчит отдельно — должна быть внутри dropdown.

### Главная (Landing)
- Dropzone слишком маленький — не доминирует на первом экране.
- Нет 3-шаговой подсказки прямо рядом с dropzone ("Загрузите → Выберите → Скачайте").
- Секции преимуществ уже есть, но тексты можно усилить по ТЗ.

### Burger Menu (панель действий)
- Группировка есть (special/process/convert), но заголовки групп слабые.
- Нет мини-сводки по файлу в верхней части меню.
- Нет разделения на "Быстрые действия" / "Редактирование" / "Оптимизация".

### Resize Panel
- Использует старые классы (`text-slate-500`, `border-slate-300`) вместо дизайн-системы.
- Процентные пресеты 25-200% — по ТЗ нужно убрать и заменить на селектор единиц (px/mm/cm/in).
- Подсказки по клавишам слишком простые — нужны Alt+drag, Shift+drag.

### Остальные страницы
- Profile/History/Settings используют старые классы в некоторых местах.
- Resize-panel единственная панель со старыми slate-классами.
